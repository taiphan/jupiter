# v1.12 — Issue watchers

**Status:** Shipped  
**Depends on:** v1.8 notifications API, workspace sync  
**References:** [REQUIREMENTS.md](../REQUIREMENTS.md) · [v1.8 persistence](./v1.8-persistence-requirements.md)

---

## Goals

1. Users can **watch** issues to follow activity without being assignee/reporter.
2. Watchers receive **in-app notifications** (bell feed) when others change the issue.
3. **Reporter and assignee** are watchers by default; new assignees are auto-added.

---

## Data model

| Field | Location | Notes |
|-------|----------|-------|
| `watcherIds: string[]` | Issue (Zustand + `issues.watcher_ids` jsonb) | User ids; synced via `/api/workspace` |

---

## UX

| Surface | Behavior |
|---------|----------|
| Issue panel sidebar | Watch / Unwatch; list watchers; editors add/remove others |
| Notification bell | Activity on watched issues (server + offline client derive) |
| My Work | **Watching** section for open issues user follows |

---

## Activity

- Watcher list changes log `kind: watcher` with message "Watchers updated".

---

## Ops

After upgrade:

```bash
npm run db:push:host
```

Adds `watcher_ids` column to `issues` (default `[]`).
