# Jupiter v1.11 + v2.0 — Authentication breadth & enterprise identity

**Status:** Planned  
**Targets:** v1.11.0 (breadth) · v2.0.0 (enterprise)  
**Builds on:** [v1.6 email auth](./v1.6-auth-requirements.md), [v1.7 Google Sign-In](./v1.7-google-sign-in-requirements.md), [v1.10 mail + TOTP](./v1.10-auth-security-requirements.md)  
**Market reference:** Atlassian Cloud (SAML SSO, SCIM, enforced 2FA), GitHub/Linear (social + PAT), Okta/Azure AD (OIDC/SAML)

---

## Summary

Close the gap between Jupiter’s **current** auth (email + Google + sessions) and **market-standard SaaS / enterprise** expectations:

| Track | Version | Theme |
|-------|---------|--------|
| **Breadth** | **v1.11** | More login methods, stronger account hygiene, developer API access, session controls |
| **Enterprise** | **v2.0** | SAML/OIDC SSO, directory sync (SCIM), org security policy, compliance-oriented audit |

Shipped today is documented in [REQUIREMENTS.md §7](../REQUIREMENTS.md). This spec defines what remains.

# Part A — v1.11 Auth breadth

## Goals

1. **Social login breadth** — Microsoft Entra ID (Azure AD), GitHub, Apple (same patterns as Google: server OAuth, `oauth_accounts`).
2. **Account completeness** — Google-only users can set a password; password users manage linked providers in Settings.
3. **Session hygiene** — list active sessions, revoke one, sign out everywhere.
4. **API access** — personal access tokens for scripts/CI (scoped, revocable).
5. **Abuse resistance** — optional CAPTCHA on register/login; optional Have I Been Pwned password check.

## Non-goals (v1.11)

- SAML / OIDC federation (v2.0)
- SCIM
- WebAuthn (v2.1 stretch)
- Org-wide policy engine (v2.0)

## FR-B1 — Additional OAuth providers

| ID | Requirement |
|----|-------------|
| FR-B1.1 | Extend `oauth_accounts.provider` enum: `google` \| `microsoft` \| `github` \| `apple`. |
| FR-B1.2 | Env per provider: `AUTH_{PROVIDER}_ENABLED`, `{PROVIDER}_CLIENT_ID`, `{PROVIDER}_CLIENT_SECRET`, redirect `{APP_URL}/api/auth/{provider}/callback`. |
| FR-B1.3 | Shared module pattern: `src/server/auth/oauth/{provider}.ts` + generic `linkOrCreateOAuthUser`. |
| FR-B1.4 | **Microsoft:** OAuth 2.0 v2, scopes `openid profile email`; tenant `common` or `AUTH_MICROSOFT_TENANT_ID`. |
| FR-B1.5 | **GitHub:** scopes `read:user user:email`; require primary verified email. |
| FR-B1.6 | **Apple:** Sign in with Apple (JWT client secret rotation documented for operators). |
| FR-B1.7 | UI: provider buttons on login/signup (show only enabled providers from `GET /api/auth/config`). |
| FR-B1.8 | Settings: list linked providers; disconnect per provider (same rules as Google — must retain another login method). |
| FR-B1.9 | Link by verified email across providers (same as v1.7 Google). |

### API additions (v1.11)

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/api/auth/microsoft` | Start Microsoft OAuth |
| GET | `/api/auth/microsoft/callback` | Complete Microsoft OAuth |
| GET | `/api/auth/github` | Start GitHub OAuth |
| GET | `/api/auth/github/callback` | Complete GitHub OAuth |
| GET | `/api/auth/apple` | Start Apple OAuth |
| POST | `/api/auth/apple/callback` | Apple POST callback |
| POST | `/api/auth/{provider}/disconnect` | Unlink provider |

## FR-B3 — Session management

| ID | Requirement |
|----|-------------|
| FR-B3.1 | `GET /api/auth/sessions` — list current user’s sessions: `id`, `createdAt`, `expiresAt`, `current` (this cookie), `userAgent`, `ip` (last seen). |
| FR-B3.2 | `DELETE /api/auth/sessions/:id` — revoke one session. |
| FR-B3.3 | `POST /api/auth/sessions/revoke-all` — revoke all except current (or all including current → force re-login). |
| FR-B3.4 | Store `userAgent` + `ip` on session create (login, OAuth callback, 2FA complete). |
| FR-B3.5 | Settings → Security: “Active sessions” table. |

## FR-B4 — Personal access tokens (PAT)

| ID | Requirement |
|----|-------------|
| FR-B4.1 | Table `api_tokens`: `id`, `user_id`, `name`, `token_prefix`, `token_hash`, `scopes` (json), `expires_at`, `last_used_at`, `created_at`. |
| FR-B4.2 | `POST /api/auth/tokens` — create PAT; return **plaintext token once** (`jpt_…`). |
| FR-B4.3 | `GET /api/auth/tokens` — list (no secret). |
| FR-B4.4 | `DELETE /api/auth/tokens/:id` — revoke. |
| FR-B4.5 | Authenticate API requests: `Authorization: Bearer jpt_…` **or** session cookie (document precedence). |
| FR-B4.6 | Scopes (v1.11 minimum): `workspace:read`, `workspace:write`, `issues:read`, `issues:write` — enforce on existing `/api/*` routes incrementally. |
| FR-B4.7 | PAT requires verified email; recommend 2FA when enabled (warn in UI if 2FA off). |

| ID | Requirement |
|----|-------------|
| FR-B5.1 | `AUTH_CAPTCHA_ENABLED` — hCaptcha or Turnstile on register + login after N failures. |
| FR-B5.2 | On register/password change: optional HIBP k-anonymity range check; reject known breached passwords. |

## Acceptance — v1.11

- [ ] Sign in with Microsoft and GitHub on localhost and production.
- [ ] Link GitHub to existing email account without duplicate user.
- [ ] Google-only user sets password and can disconnect Google while keeping email login.
- [ ] User revokes another device session; stolen session cookie stops working.
- [ ] PAT can call `GET /api/workspace` with read scope; write scope required for `PUT`.

# Part B — v2.0 Enterprise identity

## Goals

1. **SAML 2.0 Service Provider** — customers connect Okta, Azure AD, Google Workspace SAML apps.
2. **OIDC** — alternative to SAML for modern IdPs (authorization code + PKCE, same session issuance).
3. **SCIM 2.0** — provision/deprovision users and group membership from IdP.
4. **Organization security policy** — enforce 2FA, restrict email domains, disable password login when SSO required.
5. **Auth audit** — immutable-style log of security events for admins (complement v1.8 workspace audit).

## Non-goals (v2.0)

- Multi-tenant billing / seat licensing (Atlassian Guard parity)
- Data residency regions
- On-prem / Data Center clustering
- Custom IdP scripting

## FR-E1 — Organization & SSO config

| ID | Requirement |
|----|-------------|
| FR-E1.1 | Table `organizations` (or `workspaces`): `id`, `name`, `slug`, `created_at`. |
| FR-E1.2 | Table `org_sso_config`: `org_id`, `protocol` (`saml` \| `oidc`), metadata (encrypted json), `enabled`, `enforce_sso`. |
| FR-E1.3 | All users belong to exactly one org (migration: default org for existing seeds). |
| FR-E1.4 | Admin Settings → **Authentication** → configure SAML metadata URL or OIDC issuer/client. |
| FR-E1.5 | `enforce_sso=true` — disable email/password and social login for org members (except break-glass admin). |
| FR-E1.6 | Break-glass: env `AUTH_BREAK_GLASS_EMAILS` comma-list can always password-login (document for incidents). |

## FR-E2 — SAML 2.0 SP

| ID | Requirement |
|----|-------------|
| FR-E2.1 | SP metadata endpoint `GET /api/auth/saml/metadata?org=slug`. |
| FR-E2.2 | `GET /api/auth/saml/login?org=slug` — AuthnRequest redirect to IdP. |
| FR-E2.3 | `POST /api/auth/saml/acs` — Assertion Consumer Service; validate signature, `NotOnOrAfter`, audience. |
| FR-E2.4 | Map `NameID` / `email` attribute to Jupiter user (JIT create or match existing). |
| FR-E2.5 | Session: same `jupiter_session` cookie after successful assertion. |
| FR-E2.6 | Single Logout (SLO) — stretch; document session-only logout for v2.0. |

**Libraries:** `@node-saml/node-saml` or equivalent; no custom XML crypto.

## FR-E3 — OIDC enterprise

| ID | Requirement |
|----|-------------|
| FR-E3.1 | Per-org OIDC: `issuer`, `client_id`, `client_secret`, `scopes` (default `openid email profile`). |
| FR-E3.2 | Routes mirror social OAuth but `org` in state: `/api/auth/oidc/:orgSlug/login`, `/callback`. |
| FR-E3.3 | Validate `iss`, `aud`, `nonce`, `email_verified` where present. |
| FR-E3.4 | Optional `hd` / tenant claim mapping for Google Workspace / Azure tenant. |

## FR-E4 — SCIM 2.0 provisioning

| ID | Requirement |
|----|-------------|
| FR-E4.1 | `POST/PATCH/DELETE /api/scim/v2/Users` — bearer token `SCIM_BEARER_TOKEN` per org (rotatable). |
| FR-E4.2 | `POST/PATCH/DELETE /api/scim/v2/Groups` — map to Jupiter roles or project membership (minimum: org role). |
| FR-E4.3 | Deprovision: deactivate user (`active=false`), revoke all sessions and PATs. |
| FR-E4.4 | IdP guides in docs: Okta, Azure AD, Google Workspace SCIM apps. |

---

## FR-E5 — Organization security policy

| ID | Requirement |
|----|-------------|
| FR-E5.1 | `org_policies` json: `require_2fa`, `allowed_email_domains[]`, `invite_only`, `session_max_age_hours`. |
| FR-E5.2 | `require_2fa` — block app access until user enrolls TOTP (grace period configurable). |
| FR-E5.3 | `allowed_email_domains` — reject register and SSO JIT if email domain not listed. |
| FR-E5.4 | `invite_only` — `POST /api/auth/register` returns 403 unless valid invite token. |
| FR-E5.5 | Table `org_invites`: email, role, token, expires_at. |
| FR-E5.6 | Admin can reset user 2FA with audit event `security.2fa_reset_by_admin`. |

## FR-E6 — Auth audit log

| ID | Requirement |
|----|-------------|
| FR-E6.1 | Table `auth_events`: `id`, `org_id`, `user_id`, `type`, `ip`, `user_agent`, `metadata` (json), `created_at`. |
| FR-E6.2 | Event types: `login.success`, `login.failure`, `logout`, `oauth.link`, `oauth.login`, `saml.login`, `oidc.login`, `2fa.enroll`, `2fa.challenge`, `2fa.disable`, `pat.create`, `pat.revoke`, `session.revoke`, `scim.user.create`, `scim.user.deactivate`, `policy.violation`. |
| FR-E6.3 | `GET /api/admin/auth-events` — paginated, admin-only, filter by user/date/type. |
| FR-E6.4 | No secrets, tokens, or passwords in `metadata`. |

## Environment variables (v2.0)

```bash
# Enterprise SSO (per deployment or per org in DB)
AUTH_SAML_ENABLED=false
AUTH_OIDC_ENABLED=false
AUTH_SCIM_ENABLED=false
AUTH_BREAK_GLASS_EMAILS=ops@company.com

# Org defaults (seed / single-tenant mode)
ORG_DEFAULT_SLUG=acme
ORG_ENFORCE_2FA=false
ORG_ALLOWED_EMAIL_DOMAINS=acme.com,acme.dev
```
